## 新增需求

### 需求:人员资料与生日管理
系统必须允许用户维护送礼对象资料，并为每个人保存姓名、生日、关系和备注信息。

#### 场景:创建送礼对象
- **当** 用户新增一个人员并填写姓名与生日
- **那么** 系统必须保存该人员资料，并允许后续将礼物记录关联到该人员

### 需求:送礼台账记录
系统必须允许用户记录送出或收到的礼物，包含关联人员、日期、方向、礼物名称、金额和备注，并以单条记录形式展示，不得强制按人累计结算。

#### 场景:记录送出的礼物
- **当** 用户为某个人新增一条“我送出”的礼物记录
- **那么** 系统必须保存该记录，并在该人员的礼物历史中按日期显示这条条目

#### 场景:记录收到的礼物
- **当** 用户为某个人新增一条“对方送我”的礼物记录
- **那么** 系统必须保存该记录，并与送出记录一起按单条条目展示

### 需求:生日提前两周生成准备礼物 Todo
系统必须在人员生日到来前两周自动生成一条“准备礼物”Todo，并将该 Todo 与对应人员关联。

#### 场景:生日前两周生成 Todo
- **当** 某个人的生日距离当前日期还有 14 天
- **那么** 系统必须自动创建一条与该人员关联的“准备礼物”Todo

#### 场景:已有待办时避免重复生成
- **当** 某个人在同一生日周期内已经存在未删除的“准备礼物”Todo
- **那么** 系统必须避免重复创建新的同类 Todo
