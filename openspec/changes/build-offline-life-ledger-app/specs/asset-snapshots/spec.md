## 新增需求

### 需求:账户档案管理
系统必须允许用户维护多个资产账户档案，并为每个账户保存名称、账户类型、完整账户号码、备注以及是否纳入总资产汇总的标记。

#### 场景:创建纳入汇总的账户
- **当** 用户新建一个账户并填写名称、类型、完整号码，且将其标记为纳入总资产
- **那么** 系统保存该账户档案，并在后续资产快照的可选账户列表中显示该账户

#### 场景:查看并复制完整号码
- **当** 用户进入某个账户的详情页
- **那么** 系统必须显示完整账户号码并提供复制操作

### 需求:自由日期资产快照
系统必须允许用户按任意过去或当前日期创建资产快照；当用户未显式选择日期时，系统必须默认使用当前日期时间。

#### 场景:补录历史快照
- **当** 用户创建资产快照并选择一个过去的日期
- **那么** 系统必须保存该快照日期，并将该记录视为历史补录而不是当前录入时间点

#### 场景:使用当前时间创建快照
- **当** 用户创建资产快照且未手动选择日期
- **那么** 系统必须自动使用当前日期时间作为快照日期

### 需求:快照必须记录余额、负债、下次记录时间和目标
系统必须允许用户在每次资产快照中记录各账户余额、可选负债信息、下次记录时间、目标总资产以及备注信息。

#### 场景:记录完整快照
- **当** 用户提交一条资产快照并填写账户余额、负债记录、下次记录时间和目标总资产
- **那么** 系统必须保存这些字段，并在查看该快照时完整展示

### 需求:总资产变化按快照日期排序计算
系统必须按照快照日期排序计算总资产变化，并且只将被标记为纳入总资产的账户余额计入总资产；负债记录禁止自动按负数并入总资产。

#### 场景:补录后重算变化
- **当** 用户新增一条日期早于最近快照的历史记录
- **那么** 系统必须基于所有快照日期重新计算总资产时间线和相邻快照变化值

#### 场景:负债不影响总资产
- **当** 用户在快照中记录了负债金额
- **那么** 系统展示的总资产必须保持为账户汇总值，不因负债记录而自动减少

### 需求:目标达成判断
系统必须允许用户为某次快照设定目标总资产，并在后续相关视图中依据“总资产大于等于目标值”判断目标是否达成。

#### 场景:目标已达成
- **当** 某次快照记录的总资产大于或等于其目标总资产
- **那么** 系统必须将该目标标记为已达成

#### 场景:目标未达成
- **当** 某次快照记录的总资产小于其目标总资产
- **那么** 系统必须将该目标标记为未达成

### 需求:周期花销分类汇总
系统必须允许用户为每次快照维护自定义花销分类汇总，分类数量最多 10 个，且每个分类仅记录该周期的汇总金额。

#### 场景:添加不超过上限的分类汇总
- **当** 用户在某次快照中录入多个花销分类汇总，且分类总数不超过 10 个
- **那么** 系统必须保存每个分类的名称和该周期汇总金额

#### 场景:超过分类上限
- **当** 用户尝试添加第 11 个花销分类
- **那么** 系统必须阻止保存并提示分类上限为 10 个

### 需求:提供账户级对比入口
系统必须以总资产变化作为主视图，同时提供账户级余额对比的次级入口。

#### 场景:查看账户级对比
- **当** 用户从总资产变化页面进入账户对比入口
- **那么** 系统必须展示各账户在相邻快照之间的余额变化
