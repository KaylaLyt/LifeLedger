## 新增需求

### 需求:应用级密码锁
系统必须在进入应用关键内容前验证本地密码，并记录连续失败次数。

#### 场景:密码验证成功
- **当** 用户输入正确密码
- **那么** 系统必须允许其进入应用，并将连续失败次数重置为 0

#### 场景:密码验证失败
- **当** 用户输入错误密码
- **那么** 系统必须拒绝进入应用，并增加连续失败次数

### 需求:失败销毁前必须明确警告
系统必须在达到数据销毁阈值前明确提示剩余失败次数，并在最后一次允许尝试前发出不可恢复警告。

#### 场景:倒数第二次失败后警告
- **当** 用户已经连续 4 次输入错误密码
- **那么** 系统必须明确提示“再失败 1 次将销毁本地数据”

### 需求:连续五次失败销毁应用内数据
系统必须在连续 5 次密码失败后销毁应用内本地数据，但不得删除此前已经导出的 JSON 文件。

#### 场景:第五次失败触发销毁
- **当** 用户连续第 5 次输入错误密码
- **那么** 系统必须清除应用内部数据库和本地设置数据

#### 场景:已导出文件不受影响
- **当** 系统执行本地数据销毁
- **那么** 系统不得删除用户先前导出到应用外部位置的 JSON 文件

### 需求:导出单个 JSON 数据包
系统必须允许用户将应用内数据导出为单个 JSON 文件，至少包含 `schemaVersion`、`exportedAt`、`accounts`、`snapshots`、`people`、`giftRecords` 和 `todos` 字段。

#### 场景:成功导出
- **当** 用户执行数据导出
- **那么** 系统必须生成一个单独的 JSON 文件，并包含当前应用内的全部核心数据

#### 场景:导出结构带版本信息
- **当** 系统写出 JSON 文件
- **那么** 根对象必须包含 `schemaVersion` 和 `exportedAt` 字段以支持后续兼容
