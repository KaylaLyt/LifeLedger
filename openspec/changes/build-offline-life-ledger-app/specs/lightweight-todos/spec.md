## 新增需求

### 需求:轻量待办管理
系统必须允许用户创建、查看、完成和重新打开轻量 Todo，并为每条 Todo 保存标题、说明、状态、提醒时间和完成时间等基本信息。

#### 场景:创建单次 Todo
- **当** 用户新增一条不带循环规则的 Todo
- **那么** 系统必须将其保存为单次任务，并在列表中显示其基本状态信息

#### 场景:完成已存在的 Todo
- **当** 用户将某条未完成 Todo 标记为完成
- **那么** 系统必须保存其完成状态和完成时间

### 需求:常见循环规则
系统必须为 Todo 提供常见循环规则配置，至少覆盖按天、按周、按月等常见周期。

#### 场景:配置每周重复
- **当** 用户为某条 Todo 选择每周重复
- **那么** 系统必须保存该循环规则，并在完成后继续计算下一次待执行时间

### 需求:高级循环规则通过表单生成
系统必须提供高级循环规则表单，以生成更灵活的重复规则，并禁止要求用户直接输入 cron 表达式文本。

#### 场景:使用高级表单配置循环
- **当** 用户进入高级模式并通过表单设置多个时间维度
- **那么** 系统必须保存由表单生成的循环规则，并展示下一次触发时间预览

#### 场景:不暴露原始 cron 文本输入
- **当** 用户配置高级循环规则
- **那么** 系统不得要求用户直接输入原始 cron 表达式

### 需求:本地提醒与基本信息查看
系统必须基于 Todo 的提醒时间或下一次触发时间安排本地提醒，并提供轻量的基本信息视图，而不是复杂统计面板。

#### 场景:到达提醒时间
- **当** 某条 Todo 到达提醒时间或下一次触发时间
- **那么** 系统必须尝试发出本地提醒通知

#### 场景:查看 Todo 基本信息
- **当** 用户进入 Todo 列表或详情
- **那么** 系统必须展示标题、状态、下一次提醒时间和最近完成时间等基本信息
