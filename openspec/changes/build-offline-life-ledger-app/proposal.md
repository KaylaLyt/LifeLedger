## 为什么

需要一个仅供个人或家庭成员各自在本地使用的 Android 应用，用于长期保存资产快照、送礼台账和轻量待办信息。现有通用记账或待办产品通常偏向高频流水、联网同步或复杂协作，不适合以“隔数月全量盘点一次、长期本地保存、可导出可迁移”为核心的使用方式。

## 变更内容

- 新增一个离线优先的个人生活账本应用，覆盖资产快照、人情往来和轻量 Todo 三类核心场景。
- 支持配置多个账户档案，保存完整账户号码，并按任意日期录入资产快照，包含账户余额、负债记录、周期花销汇总、下次记录时间和目标总资产。
- 支持补录历史快照，并以快照日期而非录入时间计算总资产变化和目标达成情况。
- 支持自定义最多 10 个花销分类，仅记录每个周期内各分类汇总值，不引入高频流水账模型。
- 新增送礼台账，支持维护人员资料、生日日期以及收礼/送礼记录；在生日提前两周自动生成“准备礼物”Todo。
- 新增轻量 Todo，支持单次任务、常见周期和通过表单生成的高级循环规则，并提供基本列表与状态信息。
- 新增本地安全与数据迁移能力，包括界面密码锁、密码失败预警、连续 5 次失败销毁应用内数据，以及导出单个 JSON 数据包。

## 功能 (Capabilities)

### 新增功能
- `asset-snapshots`: 账户管理、自由日期资产快照、补录历史、目标设定与总资产波动查看。
- `gift-ledger`: 人员资料、生日管理、送礼台账以及生日衍生待办生成。
- `lightweight-todos`: 轻量待办、循环规则、本地提醒和基础状态查看。
- `local-security-and-export`: 本地密码锁、失败销毁预警、应用内数据清理和 JSON 导出。

## 影响

- 需要定义 Android 本地数据模型、数据库结构和 JSON 导出格式。
- 需要设计跨模块联动逻辑，例如生日自动生成 Todo、快照目标达成判断和密码失败次数处理。
- 需要处理本地提醒、敏感信息展示与复制、补录排序以及应用内数据销毁策略。
