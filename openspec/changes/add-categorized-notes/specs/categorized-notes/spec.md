## 新增需求

### 需求:系统必须提供分类列表首页
系统必须提供一个以分类列表为首页的分类笔记入口，并展示未分类入口和每个真实分类对应的笔记数量。

#### 场景:首页展示未分类和真实分类
- **当** 用户进入分类笔记模块
- **那么** 系统必须先展示分类列表首页，并显示未分类入口以及所有真实分类

#### 场景:分类显示笔记数量
- **当** 系统渲染分类列表首页
- **那么** 每个分类入口都必须显示该分类当前包含的笔记数量

### 需求:未分类必须作为虚拟分类存在
系统必须将未分类视为虚拟分类入口，而不是真实分类记录；未归类的笔记必须通过该入口浏览，且该入口禁止被删除。

#### 场景:笔记未归类时进入未分类
- **当** 一条笔记没有关联任何真实分类
- **那么** 系统必须在未分类入口下展示该笔记

#### 场景:未分类不可删除
- **当** 用户查看分类管理界面
- **那么** 系统必须禁止删除未分类入口

### 需求:系统必须支持唯一分类名和临时创建分类
系统必须允许用户创建单层真实分类，且分类名必须唯一；在保存笔记时，如果用户输入新的唯一分类名，系统必须允许即时创建该分类。

#### 场景:创建唯一分类成功
- **当** 用户输入一个当前不存在的分类名并确认创建
- **那么** 系统必须新增该真实分类

#### 场景:重复分类名被拒绝
- **当** 用户尝试创建与现有真实分类同名的分类
- **那么** 系统必须拒绝创建并提示分类名重复

#### 场景:保存笔记时临时创建分类
- **当** 用户在保存笔记时选择创建新的唯一分类名
- **那么** 系统必须先创建该分类，再将笔记归入该分类

### 需求:真实分类必须支持置顶新增和拖动排序
系统必须让真实分类默认按新建时间倒序显示，新建分类必须放在最上面；当用户拖动调整真实分类顺序后，系统必须持久化该顺序。

#### 场景:新建分类显示在最上面
- **当** 用户成功创建一个新的真实分类
- **那么** 系统必须在分类列表中将该分类显示在所有真实分类的最上方

#### 场景:拖动排序后保持顺序
- **当** 用户拖动调整真实分类顺序并返回分类列表
- **那么** 系统必须继续按用户调整后的顺序展示这些真实分类

### 需求:系统必须支持无标题纯文本笔记
系统必须允许用户保存只有正文的纯文本多行笔记，且禁止保存空内容笔记。

#### 场景:保存纯文本笔记
- **当** 用户输入非空正文并保存笔记
- **那么** 系统必须保存该笔记正文，而不要求标题字段

#### 场景:空内容笔记保存失败
- **当** 用户尝试保存空正文或仅包含空白字符的笔记
- **那么** 系统必须拒绝保存该笔记

### 需求:分类内笔记必须按更新时间倒序浏览
系统必须允许用户进入任意分类或未分类入口查看笔记，并按更新时间倒序展示笔记列表；列表必须展示正文前段内容作为预览。

#### 场景:分类内按更新时间倒序显示
- **当** 用户进入某个真实分类
- **那么** 系统必须按更新时间从新到旧展示该分类下的笔记

#### 场景:列表显示正文预览
- **当** 系统展示笔记列表项
- **那么** 系统必须显示该笔记正文前段内容作为列表预览

### 需求:删除真实分类时笔记必须回退到未分类
系统必须允许用户删除真实分类，并在删除后将该分类下的所有笔记保留并回退到未分类。

#### 场景:删除分类后保留笔记
- **当** 用户删除一个真实分类
- **那么** 系统必须移除该分类，并将原属于该分类的笔记改为未分类

### 需求:系统必须支持单个分类内批量删除笔记
系统必须允许用户在当前分类内进入批量选择模式，并一次删除多条已选笔记；该批量删除范围必须仅限当前分类。

#### 场景:在当前分类中批量删除
- **当** 用户在某个分类的笔记列表中选择多条笔记并确认删除
- **那么** 系统必须删除这些已选笔记

#### 场景:批量删除范围不跨分类
- **当** 用户在某个分类的批量选择模式中执行删除
- **那么** 系统必须只允许删除当前分类列表中的笔记

### 需求:分类和笔记数据必须纳入单文件 JSON 导出
系统必须在单文件 JSON 导出中包含真实分类和笔记数据，并保留已有根对象的版本信息。

#### 场景:导出包含分类和笔记
- **当** 用户执行应用数据导出
- **那么** 系统必须在导出的 JSON 中包含分类数据和笔记数据

#### 场景:未分类笔记导出为无分类关联
- **当** 导出一条未归类的笔记
- **那么** 系统必须在导出结果中将该笔记表示为没有真实分类关联
