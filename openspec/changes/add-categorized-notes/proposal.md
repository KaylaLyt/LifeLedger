## 为什么

当前“随时想法”的记录方式主要依赖微信收藏，虽然录入门槛低，但分类和后续整理能力较差，导致内容积累后难以按主题回看。现有应用已经覆盖资产、送礼和 Todo，但还缺少一个适合长期保存纯文本想法、并以分类浏览为核心的轻量笔记能力。

## 变更内容

- 新增一个离线优先的分类笔记模块，用于记录无标题、纯文本、多行内容的随时想法。
- 支持单层分类管理，分类名必须唯一，可在保存笔记时临时创建新分类。
- 支持“未分类”视图；未分类为虚拟分类，不作为真实分类记录存在，也不可删除。
- 支持分类列表首页展示每个分类的笔记数量；真实分类默认按创建时间倒序排列，新建分类置顶，并支持拖动后持久化自定义顺序。
- 支持分类内笔记列表按更新时间倒序浏览，列表仅展示正文前段内容，不引入标题字段。
- 支持将笔记移动到其他分类、删除真实分类时将其下所有笔记自动回退到未分类，以及在单个分类内执行批量删除笔记。
- 保持单文件 JSON 导出能力，将分类和笔记数据纳入导出结果。

## 功能 (Capabilities)

### 新增功能
- `categorized-notes`: 单层分类、无标题纯文本笔记、分类浏览、批量删除和分类排序管理。

## 影响

- 需要新增分类与笔记的数据模型、查询结构以及排序字段。
- 需要扩展首页导航与分类/笔记管理界面，支持拖动排序与分类内批量选择删除。
- 需要更新 JSON 导出结构，将分类和笔记内容纳入单文件导出结果。
